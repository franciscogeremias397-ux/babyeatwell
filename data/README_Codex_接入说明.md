# 宝贝乖乖吃饭 · 食谱数据包 v1.0

本数据包用于修复小程序中“食材用量混乱、制作步骤重复模板化”的问题。

## 文件清单

- `age_rules.json`：12-18 月龄、19-36 月龄规则。
- `recipes.json`：80 道完整食谱数据。所有食材用量、制作步骤、营养亮点、注意事项都必须从这里读取。
- `ingredient_tags.json`：标准食材标签库。
- `tag_alias_map.json`：用户输入别名、错别字、地方叫法到标准标签的映射。
- `image_prompts.csv`：80 道食谱的图片生成提示词。
- `recipes_validation_report.json`：数据校验报告。

## 给 Codex 的强制接入要求

1. 一周食谱和随机一餐只负责筛选、抽取 `recipes.json` 中已有食谱，不得临时编造食谱。
2. 食材用量必须直接读取 `recipe.ingredients`，不得动态生成克数，不得按模板随机改写。
3. 制作步骤必须直接读取 `recipe.steps`，不得使用统一模板生成“清洗、切碎、煮熟”之类的重复步骤。
4. 营养亮点必须直接读取 `recipe.nutritionHighlights`。
5. 注意事项必须直接读取 `recipe.cautions`。
6. 过敏/忌口过滤必须使用 `recipe.allergens + recipe.avoidTags` 与宝宝档案标准标签比对。
7. 如果经过过滤后食谱池不足，必须提示“可用食谱不足”，不得临时编造新食谱。
8. 食谱图片展示必须通过 `utils/media.js` 的 `recipeImage(recipe)` 获取。80 张正式图上传到云存储后，统一在媒体配置里切换；不要把云端域名或云文件 ID 分散写进页面。

## 宝宝档案要求

宝宝档案只保存 `birthDate`，不保存固定月龄。系统每次进入首页、生成一周食谱或随机一餐时，都应根据当前日期实时计算月龄。

## 数据边界说明

本数据包提供的是家庭饮食参考数据，不是医疗营养处方。页面和下载卡片中应保留提示：食材用量为家庭参考量，可根据宝宝食欲、咀嚼能力和实际进食情况微调；如宝宝存在过敏、疾病或特殊营养需求，请咨询医生或营养师。
